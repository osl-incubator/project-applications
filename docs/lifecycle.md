# Project Lifecycle

The OSL Incubator Program uses a staged lifecycle so applicants, maintainers,
reviewers, and contributors understand what support is available and what
evidence is expected.

```text
Proof of Concept (PoC) -> Incubation -> Graduated
```

A project may also become inactive, paused, removed from public lists, or
archived after maintenance review.

## Proof of Concept (PoC)

**Purpose:** validate feasibility, usefulness, and OSL alignment.

PoC is appropriate for ideas, proposals, prototypes, or restarting efforts that
need a short exploration period before full incubation.

Expected evidence:

- problem statement;
- intended users or beneficiaries;
- proposal, prototype, or early repository;
- reachable maintainer;
- public repository or proposal repository;
- license;
- Code of Conduct;
- initial tasks or validation questions;
- 1-3 month exploration plan;
- expected OSL support.

Exit outcomes:

- move to Incubation;
- continue PoC with revised milestones;
- pause;
- decline or end review.

## Incubation

**Purpose:** structured development and sustainability.

Incubation is for projects ready to work on milestones, maintenance practices,
contributor readiness, documentation, quality, governance, and review.

Expected evidence:

- active maintainer participation;
- roadmap or milestones;
- public issues, tasks, or project board;
- contribution instructions;
- documentation plan;
- tests or quality plan;
- release or publication plan where relevant;
- mentor/reviewer communication;
- scientific review plan when relevant;
- OSL status acknowledgement in README/docs.

Exit outcomes:

- graduate;
- continue incubation with new milestones;
- pause;
- archive or remove after human review.

## Graduated

**Purpose:** project has reached a sustainable state and can operate with less
direct OSL incubation support.

Expected evidence:

- milestones completed;
- active or stable maintenance;
- governance or maintainer model;
- release, adoption, citation, deployment, or user evidence where available;
- contributor process;
- documentation;
- security reporting when relevant;
- pyOpenSci, rOpenSci, or equivalent review when required;
- repository location after graduation;
- updated status acknowledgement.

Graduation does not mean OSL guarantees future maintenance, funding, slots, or
contributors.

## Inactive / Archived

**Purpose:** prevent OSL from directing contributors to abandoned, unreachable,
closed, or unsafe projects.

Expected evidence before marking inactive or archived:

- maintenance review issue;
- maintainer pings;
- evidence of unresolved critical concern;
- human decision.

Possible outcomes:

- project remains active after remediation;
- project is paused or marked at risk;
- project is removed from public active lists;
- OSL-hosted repository is archived after review;
- project is eligible for reactivation after criteria are restored.

## Reactivation

A paused, inactive, or archived project may request reactivation when:

- maintainers request reactivation;
- baseline criteria are restored;
- license and Code of Conduct are present;
- repository is public and reachable;
- maintainers provide a maintenance plan;
- roadmap or milestones are current;
- OSL review approves the requested stage.
