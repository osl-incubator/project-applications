# Scientific Software Review

Scientific software often needs review beyond the basic incubation criteria.
External or equivalent review helps users and contributors trust that a project
has appropriate packaging, documentation, testing, sustainability, and domain
review practices.

For scientific Python projects in pyOpenSci scope, OSL graduation depends on
acceptance by pyOpenSci. For scientific R projects in rOpenSci scope, OSL
graduation depends on acceptance by rOpenSci. Projects outside those scopes
should agree with OSL on an equivalent review path during incubation, not at the
last minute before graduation.

## Out-of-scope scientific projects

Projects outside pyOpenSci/rOpenSci scope should work with OSL reviewers to
define an equivalent review path. Equivalent review may include domain experts,
community review, journal/software review, reproducibility checks, packaging and
testing review, or another documented process appropriate to the project.

## When to define the review path

Define the likely review path during PoC or early incubation. Waiting until a
graduation request can delay graduation and create unclear expectations.

## Evidence to submit for graduation

Graduation requests should include:

- review path type: pyOpenSci, rOpenSci, or equivalent;
- review status;
- acceptance URL or review issue URL when available;
- unresolved review blockers;
- reviewer or community feedback summaries where appropriate;
- explanation if the project is not in scope for external review.
