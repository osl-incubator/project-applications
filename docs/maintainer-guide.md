# Maintainer Guide

Maintainers remain responsible for day-to-day project maintenance, roadmap
decisions, releases, contributor review, and public communication during OSL
incubation.

## Responsibilities

Maintainers should:

- maintain an OSI-approved open-source license;
- keep a public `LICENSE` file available;
- maintain a Code of Conduct with a reporting path;
- keep the public repository accessible;
- respond to OSL review pings;
- keep roadmap and milestones current;
- communicate pauses, maintainer changes, and capacity limits;
- review contributor work when inviting contributors;
- keep README/docs acknowledgement accurate;
- avoid implying OSL ownership, certification, guarantee, or indefinite
  maintenance;
- notify OSL about repository transfer, archive, relicensing, major scope
  changes, or security reporting changes;
- maintain a security reporting path when relevant.

## Communication with OSL

Keep review issues updated with current links and context. If maintainers become
unavailable, post a pause plan or identify another maintainer. Silence can
trigger maintenance review and at-risk labels.

## Contributor readiness

Projects inviting contributors should provide:

- contribution instructions;
- public issue tracker or project board;
- starter issues or scoped tasks;
- expected development setup;
- review expectations;
- communication channels;
- Code of Conduct and reporting path.

## GSoC and internship readiness

Projects interested in GSoC, internships, grants, or similar limited programs
should prepare:

- mentors and backup reviewers;
- scoped project ideas;
- expected outcomes;
- contributor requirements;
- public issue links;
- communication channel;
- review cadence;
- capacity to support contributors.

OSL participation, external selection, contributor slots, funding, and project
idea acceptance are not guaranteed.

## Status acknowledgement

Accepted projects should add status acknowledgement text to `README.md` and
public documentation where applicable. Update the text when the project moves
from PoC to Incubation, graduates, pauses, becomes at risk, moves outside OSL,
or is archived/removed from active OSL lists.

See [acknowledgement.md](acknowledgement.md) for examples.
